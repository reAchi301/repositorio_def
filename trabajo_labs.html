¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es la infraestructura global que conecta millones de dispositivos a través de redes de comunicaciones, mientras que la World Wide Web (WWW) es un sistema de acceso a la información que se encuentra en Internet, usando el protocolo HTTP para navegar por sitios web.

¿Cuáles son las partes de un URL?

Un URL (Uniform Resource Locator) tiene varias partes:
Esquema (protocolo): http, https, ftp, etc.
Dominio (host): ej. www.ejemplo.com
Puerto (opcional): ej. :80 (puerto estándar para HTTP)
Ruta: ej. /pagina1
Consulta (opcional): ej. ?id=123
Fragmento (opcional): ej. #seccion

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET: Solicitar datos desde el servidor.
HEAD: Similar a GET, pero solo obtiene los encabezados de respuesta, sin el cuerpo.
POST: Enviar datos al servidor (generalmente para crear o actualizar recursos).
PUT: Sustituir un recurso completo en el servidor.
PATCH: Actualizar parcialmente un recurso en el servidor.
DELETE: Eliminar un recurso del servidor

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Se utiliza POST porque este método permite enviar datos de manera segura (en el cuerpo de la solicitud),
lo que es adecuado para la transmisión de información sensible, como un usuario y contraseña.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Se utiliza el método GET para solicitar la página desde el servidor.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

El código 200 significa que la solicitud se ha procesado correctamente y no ha ocurrido ningún error. Es una respuesta exitosa.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Sí, es responsabilidad del desarrollador corregirlo, ya que el error 404 indica que la página solicitada no se encuentra disponible. El desarrollador debe asegurarse de que las URL sean correctas y de que las páginas o recursos estén accesibles.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Sí, es responsabilidad del desarrollador. El error 500 es un error del servidor, lo que significa que algo falló en la aplicación o en el servidor. El desarrollador debe investigar y corregir el problema.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Un atributo deprecated significa que ya no se recomienda su uso y puede ser eliminado en futuras versiones de HTML. Ejemplos de elementos de HTML 4 desaprobados en HTML5 incluyen: font, center, big, y strike.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

HTML5 introduce nuevas etiquetas semánticas como article, section, nav, y footer Además, mejora el soporte multimedia con video y audio, elimina elementos obsoletos y tiene una mejor integración con JavaScript y APIs, además de mayor soporte para aplicaciones móviles.

¿Qué componentes de estructura y estilo tiene una tabla?

Estructura: table, tr (fila), th (encabezado de celda), td (celda de datos), caption (título de la tabla).
Estilo: Las tablas pueden estilizarse con CSS, afectando el borde, el fondo, el tamaño de las celdas, etc.

¿Cuáles son los principales controles de una forma HTML5?

Algunos controles principales son: input: para campos de texto, contraseñas, fechas, correos electrónicos, etc. textarea: para texto largo. select: lista desplegable. button: botones interactivos. input type="submit": para enviar el formulario.

Sobre el ciclo de vida y desarrollo de los sistemas de información:

¿Cuál es el ciclo de vida de los sistemas de información?

El ciclo de vida de los sistemas de información generalmente incluye las siguientes fases:
Planificación
Análisis de requisitos
Diseño
Desarrollo
Implementación
Mantenimiento

¿Cuál es el ciclo de desarrollo de sistemas de información?

El ciclo de desarrollo sigue un proceso similar, pero enfocado más en la construcción y despliegue del sistema. Incluye:
Planificación
Diseño
Desarrollo
Pruebas
Implementación
Mantenimiento y actualización.
